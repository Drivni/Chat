# P2P LAN Chat

Консольный peer-to-peer мессенджер для локальной сети.  
Обнаружение узлов — UDP broadcast, обмен сообщениями — TCP.

---

## Запуск

```bash
python chat.py <ip_address> <username>

# Примеры (три отдельных терминала):
python chat.py 127.0.0.1 Alice
python chat.py 127.0.0.2 Bob
python chat.py 127.0.0.3 Carol
```

### Команды в чате

| Команда  | Действие                        |
|----------|---------------------------------|
| `/peers` | Показать список активных узлов  |
| `/quit`  | Выйти из чата                   |

---

## Тестирование через Docker

```bash
# Запустить три узла одновременно (каждый в своём контейнере):
docker compose up

# Подключиться к конкретному экземпляру:
docker compose exec alice bash   # затем python chat.py 127.0.0.1 Alice
```

Или запустить по одному вручную:

```bash
docker run --rm -it --network host -v $(pwd):/app -w /app python:3.11-slim \
    python chat.py 127.0.0.1 Alice

# В другом терминале:
docker run --rm -it --network host -v $(pwd):/app -w /app python:3.11-slim \
    python chat.py 127.0.0.2 Bob
```

## Архитектура

```
                    UDP broadcast (255.255.255.255:55555)
                    ┌─────────────────────────────┐
                    │  «Я появился, меня зовут X»  │
                    └────────────┬────────────────┘
                                 │  все слышат
                    ┌────────────▼────────────────┐
                    │  Каждый слышащий узел        │
                    │  → устанавливает TCP-соед.  │
                    │  → отправляет своё имя       │
                    └────────────────────────────┘

           TCP (localhost:55556) — постоянные двусторонние соединения

   Alice ──────────── Bob
     │  \           /  │
     │   \         /   │
     │    \       /    │
     │     Carol ──────┘
```

### Форматирование пакетов (бинарный фрейм)

```
Offset  Размер  Описание
──────  ──────  ──────────────────────────────────────────
0       1 байт  Тип сообщения
                  1 = MSG_CHAT   (текстовое сообщение)
                  2 = MSG_NAME   (передача имени узла)
                  4 = MSG_LEAVE  (корректное отключение)
1–2     2 байта Длина payload (big-endian uint16)
3…N     N байт  Payload (UTF-8)
```

### Поток событий — подключение нового узла

```
NewNode                 ExistingNode
  │                          │
  │──UDP bcast(MSG_NAME)────>│
  │                          │
  │<─TCP connect─────────────│
  │<─MSG_NAME(name)──────────│
  │──MSG_NAME(myname)───────>│
  │                          │
  │  [постоянный TCP канал]  │
  │<───────MSG_CHAT─────────>│
  │                          │
  │──MSG_LEAVE──────────────>│ (при /quit)
  │                          │
```

### Потоки выполнения

| Поток               | Описание                                          |
|---------------------|---------------------------------------------------|
| `main`              | Чтение ввода пользователя, отправка сообщений     |
| `_udp_listen_loop`  | Обнаружение новых узлов через UDP broadcast       |
| `_accept_loop`      | Принятие входящих TCP-соединений                  |
| `_peer_recv_loop`   | По одному потоку на каждое соединение с пиром     |
